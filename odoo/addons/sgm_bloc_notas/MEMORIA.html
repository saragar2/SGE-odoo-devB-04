1.1 Descripción de las Clases

Clase Nombre Técnico Descripción
Note sgm.note Modelo principal que representa una nota compartida. Hereda de mail.thread y mail.activity.mixin para soporte de mensajería y actividades.

1.2 Relaciones entre Modelos

Origen Tipo Destino Descripción
sgm.category One2many (1:N) sgm.note Una categoría puede contener múltiples notas

sgm.note Many2many (N:M) sgm.tag Las notas pueden tener múltiples etiquetas y viceversa
sgm.note One2many (1:N) sgm.note_line Una nota puede contener múltiples líneas/tareas

2. Finalidad y Funcionamiento del Módulo

2.1 Finalidad

El módulo SGM Bloc de Notas es una aplicación para Odoo 17 que permite a los usuarios crear, organizar y compartir notas de forma sencilla. Está diseñado para entornos empresariales donde se necesita gestionar información de manera colaborativa.

Los objetivos principales del módulo son:

  • Proporcionar un sistema centralizado para la gestión de notas
  • Permitir la organización mediante categorías y etiquetas
  • Facilitar la colaboración entre usuarios mediante notas públicas
  • Integrar funcionalidades de seguimiento y mensajería de Odoo

2.2 Funcionalidades Principales

Gestión de Notas

  • Crear notas con título, contenido HTML enriquecido e imagen representativa
  • Marcar notas como públicas o privadas
  • Seguimiento automático de la última actualización
  • Validación del título (mínimo 3 caracteres)

Organización

  • Clasificar notas por categorías (Personal, Trabajo, Estudios, etc.)
  • Etiquetar notas con múltiples tags de colores
  • Crear listas de tareas dentro de cada nota (líneas)
  • Marcar tareas como completadas

Colaboración

  • Sistema de mensajería integrado (chatter)
  • Seguimiento de actividades
  • Asignación de propietario a cada nota
  • Tracking de cambios en campos importantes

Seguridad

  • Permisos diferenciados para usuarios internos (lectura, escritura, creación)
  • Permisos completos (CRUD) para administradores

2.3 Estructura de Menús

Notas compartidas (menú raíz) ├── Notas → Lista y formulario de notas ├── Categorías → Gestión de categorías └── Etiquetas → Gestión de tags

2.4 Flujo de Uso

  1. El usuario accede al menú "Notas compartidas"
  2. Crea categorías y etiquetas para organizar sus notas
  3. Crea una nueva nota asignando categoría, etiquetas e imagen
  4. Añade líneas (tareas) a la nota
  5. Marca las tareas como completadas según avanza
  6. Opcionalmente comparte la nota marcándola como pública

Reutilización de código: La herencia de mixins como mail.thread y mail.activity.mixin permite añadir funcionalidades complejas (mensajería, actividades, seguimiento) con mínimo esfuerzo de desarrollo.

Desarrollo rápido: Una vez comprendida la arquitectura MVC de Odoo, el desarrollo de nuevos modelos y vistas es relativamente ágil gracias a las convenciones del framework.

4. Problemas Encontrados

4.1 Vistas Anidadas para Many2many

Problema: Los tests requerían una vista <tree> anidada dentro de los campos Many2many, pero el widget many2many_tags oculta visualmente esta estructura.
Solución: Añadir la vista tree dentro del campo aunque el widget la sobreescriba visualmente:
<field name="tag_ids" widget="many2many_tags"> <tree> <field name="name"/> </tree> </field>

4.2 Propietario por Defecto (OdooBot)

Problema: Al cargar datos demo, el campo owner_id tomaba el usuario del sistema (OdooBot) en lugar de un usuario real.
Causa: El valor default=lambda self: self.env.user se evalúa durante la carga de datos, cuando el usuario activo es OdooBot.

Solución: Se puede especificar explícitamente el usuario en el demo.xml mediante ref="base.user_admin" o aceptar este comportamiento por defecto.

4.3 Código Comentado en Ficheros Scaffold

Problema: Los ficheros generados por el comando scaffold (models.py, controllers.py, views.xml, templates.xml) contenían código de ejemplo comentado que hacía fallar los tests de calidad.
Solución: Limpiar todos los ficheros eliminando el código comentado y dejando solo la estructura mínima necesaria (cabecera y etiquetas vacías).

4.4 Permisos Diferenciados

Problema: Inicialmente todos los usuarios tenían permisos CRUD completos, incumpliendo el requisito de permisos diferenciados.
Solución: Crear dos líneas de permisos por modelo en ir.model.access.csv:
  • base.group_user: Leer, Escribir, Crear (sin Borrar)
  • base.group_system: CRUD completo

5. Sugerencias de Mejora

5.1 Funcionalidades Adicionales

Mejora Descripción
Compartir con usuarios específicos Implementar un campo Many2many con res.users para seleccionar con quién compartir cada nota privada.
Recordatorios Integrar con el sistema de actividades de Odoo para programar recordatorios automáticos sobre notas.
Búsqueda avanzada Añadir filtros y agrupaciones predefinidas (por categoría, por propietario, notas públicas/privadas).
Vista Kanban Implementar una vista Kanban para visualizar las notas como tarjetas organizadas por categoría o estado.
Exportación Permitir exportar notas a PDF o compartirlas directamente por email.

5.2 Mejoras Técnicas

Mejora Descripción
Record Rules Implementar reglas de registro para que los usuarios solo vean sus propias notas privadas.
Campos computados adicionales Contador de líneas completadas, porcentaje de progreso de tareas.
Archivado Añadir campo active para poder archivar notas sin eliminarlas.
Ordenación manual Implementar campo sequence para ordenar líneas manualmente con drag & drop.

5.3 Mejoras de Experiencia de Usuario

  • Widget de color para tags: Mostrar los tags con sus colores asignados en la vista tree.
  • Preview de imagen: Mostrar miniaturas de las imágenes en la vista lista.
  • Atajos de teclado: Implementar acciones rápidas para crear notas desde cualquier parte del sistema.
  • Plantillas de notas: Permitir crear notas a partir de plantillas predefinidas.
